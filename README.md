# Maryland Forward Party – Civic Technology Ideas

This repository collects potential civic technology projects discussed within the Maryland Forward Party community.  
Each idea below represents a possible open-source project that volunteers could build collaboratively.

The goal is to improve civic participation, transparency, and internal organizational tools.

---

# 1. Prioritized Problems Mobile App

A mobile application for identifying and ranking the most important problems facing Maryland residents.

## Goals
- Allow users to submit problems affecting their communities
- Enable voting or ranking of issues
- Surface the highest-priority issues for discussion and policy development
- Provide visual dashboards of top concerns

## Possible Features
- Mobile-first interface
- Ranked voting or priority scoring
- Geographic tagging of issues
- Topic categories
- Integration with Forward Party platform development

## UI Prototype
https://miro.com/app/board/uXjVNVqvYdk=/?share_link_id=442935498077

# 2. Monthly Dues Ballot System

A digital ballot system to determine voluntary monthly dues or funding contributions for the organization.

## Goals
- Transparent voting on dues levels
- Secure vote collection
- Clear reporting of results

## Considerations
- Anonymous voting
- Vote verification
- Auditability

# 3. Petition Signature Prefill Mobile App

A mobile application that helps volunteers collect petition signatures more efficiently by pre-filling forms.

## Concept
Volunteers scan or enter voter information to automatically generate properly formatted petition sheets.

## Goals
- Reduce handwriting errors
- Improve petition validity rates
- Speed up signature collection

## Potential Features
- Voter lookup
- Autofill petition forms
- Export printable petition sheets
- Offline support

Reference:
https://www.notion.so/Mobile-App-to-Prefill-Signature-Petitions-see-Andy-Ellis-31e67ecedf89806b87c8c4e5cc461194?pvs=21

# 4. Voting Spreadsheet Protection Tool

Automation scripts to help manage voting spreadsheets while preventing accidental modification of protected ranges.

## Goals
- Copy spreadsheets while preserving protected ranges
- Prevent tampering with vote totals
- Maintain auditability

## Possible Implementations
- Google Sheets API script
- Spreadsheet template generator
- CLI tool for admins

Reference:
https://www.notion.so/Write-Code-to-Copy-Protecting-Ranges-for-Voting-Spreadsheet-31e67ecedf8980879392f9505875d39f?pvs=21

# 5. Platform Topic Graph Database

A graph database representing relationships between policy ideas, issues, and community priorities.

## Goals
- Map policy topics and their relationships
- Identify clusters of related issues
- Enable data-driven platform development

## Possible Technologies
- Neo4j
- GraphQL
- Knowledge graphs
- Network visualization

Reference:
https://www.notion.so/Platform-Topic-Graph-Database-31e67ecedf89805183cde8dfc34a5b9d?pvs=21

# 6. Maryland Bills Transparency Tool

A public tool that visualizes Maryland legislation and helps citizens understand bills more easily.

Example prototype:
https://codecollective.us/mdbills.html

Legislative data source:
https://mgaleg.maryland.gov/2026rs/misc/billsmasterlist/legislation.json

## Possible Features
- Bill search and filtering
- Visual bill timelines
- Topic clustering
- Legislative activity dashboards
